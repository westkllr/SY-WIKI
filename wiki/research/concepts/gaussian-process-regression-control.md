## 한 줄 요약

데이터로부터 **예측값과 함께 불확실성(신뢰구간)을 동시에 제공**하는 베이지안 비파라미터 회귀 — 모델 불확실성을 정량화해서 "불확실한 구간에서 보수적으로 제어"하는 안전한 적응 제어를 가능하게 한다.

---

## 핵심 원리

### GPR이란?

GPR은 함수 f(x)에 대한 베이지안 사전 분포를 **커널(kernel) 함수**로 정의하고, 데이터 (X, y)를 관측한 후 사후 분포를 계산하는 방법.

```
사전:    f ~ GP(0, k(x, x'))          ← 커널이 함수의 "부드러움" 인코딩
사후:    f | X, y ~ GP(μ*(x), σ²*(x))  ← 데이터 조건부 분포

예측 평균:     μ*(x*) = k*ᵀ (K + σ²I)⁻¹ y
예측 분산:     σ²*(x*) = k** - k*ᵀ (K + σ²I)⁻¹ k*
```

핵심: 예측 평균(μ*)뿐 아니라 **예측 분산(σ²*)** 도 얻는다 → 어디서 모델이 자신 있는지, 불확실한지 알 수 있음.

### 제어에서 GPR의 역할

GPR을 제어에 사용하는 방식은 크게 두 가지:

**① 불확실성 보상 (Sarkar 2026 방식)**
```
실제 시스템:  ẋ = f_nom(x, u) + d(x, u)    ← d: 알 수 없는 불확실성/외란
GPR이 학습: d(x, u) ← 잔차 데이터로 학습
제어:        u = u_nom + u_GPR              ← GPR 보상항 추가
```
물리 기반 명목 모델 + 데이터 기반 GPR 보상 → 하이브리드 모델.

**② 불확실성 인식 MPC (Cautious MPC)**
```
MPC 제약:  x ∈ X_safe    →  μ*(x) + β·σ*(x) ∈ X_safe
```
GPR 분산이 클 때(모델 불확실) → 제약을 더 보수적으로 적용 → 안전 보장.

### 주요 커널 함수
| 커널 | 특성 | 적합 시스템 |
|------|------|-----------|
| RBF (가우시안) | 무한 미분가능, 매우 부드러움 | 연속 비선형 시스템 |
| Matérn 3/2 | 1회 미분가능 | 차량 동역학 (충격 포함) |
| 선형 커널 | 선형 GP (Bayesian linear regression) | 단순 파라메트릭 불확실성 |

## 교수님 연구 적용 방안

### 연구 영역 1 — AI 기반 차량 모델링

| 시나리오 | GPR 활용 |
|---------|---------|
| **타이어-노면 마찰 모델** | Magic Formula 명목 모델 위에 GPR 잔차 학습 → 노면 변화(건조/습윤/결빙)에 적응 |
| **배터리 내부 저항 모델** | 온도·SOC에 따른 내부 저항 변화를 GPR으로 보상; 노화 특성도 잔차로 포착 가능 |
| **모터 손실 모델** | 전동기의 비선형 효율 맵을 GPR으로 보완; 온도 의존성 자동 학습 |

### 연구 영역 4 — 전동화 파워트레인 최적 제어

GPR 불확실성 보상 + MPC = **안전한 에너지 관리**:
- 명목 에너지 관리 MPC: 배터리-모터 물리 모델 기반 QP
- GPR 보상: 배터리 노화, 온도 효과 등 모델 오차를 잔차로 학습
- 결과: 실제 에너지 소비가 예측을 벗어날 위험 감소

### 연구 영역 3 — 안전·동역학 제어

회생제동 중 타이어 슬립 임계값은 노면 조건에 따라 달라진다. GPR이 노면 마찰 불확실성을 모델링하면, **마찰 불확실성을 고려한 보수적 회생제동 토크 제한** 가능. [[adaptive-partitioning-ev-chassis]]의 파티션 내부 모델에도 적용 가능.

## 기존 방법론과 비교

| 항목 | **물리 기반 모델 + MPC** (기존) | **신경망 + MPC** | **GPR + MPC** |
|------|--------------------------|----------------|--------------|
| 모델 출처 | 제1원리 방정식 | 데이터 (블랙박스) | 데이터 (베이지안) |
| 불확실성 정량화 | 없음 (파라미터 고정) | 없음 (점 추정) | **있음 (σ²* 제공)** |
| 데이터 필요량 | 거의 없음 | 많음 | **중간** |
| 계산 복잡도 | 낮음 | 낮음 (추론) | **O(n³) 학습, O(n) 추론** |
| 외삽 성능 | 좋음 (물리 기반) | 나쁨 (과신) | **보수적 (σ* 증가로 경고)** |
| 적응성 | 낮음 (파라미터 고정) | 재학습 필요 | **온라인 업데이트 가능** |
| 해석 가능성 | 높음 | 낮음 | **중간 (커널 해석 가능)** |
| 안전성 보장 | 물리 제약 | 어려움 | **σ* 기반 보수적 제약 가능** |

> **핵심 차이 (vs. 기존 MPC)**: 기존 MPC는 모델이 맞는다고 가정하고 최적화. GPR-MPC는 모델이 얼마나 불확실한지 알고, **불확실한 곳에서 더 보수적으로 행동**한다.

### EKF와 비교 (추정 맥락에서)

| 항목 | EKF | GPR |
|------|-----|-----|
| 모델 구조 | 파라메트릭 (물리 방정식) | 비파라메트릭 |
| 노이즈 가정 | 가우시안 | 없음 (kernel로 유연성) |
| 온라인 업데이트 | 자연스러움 (재귀 구조) | Sparse GPR로 가능하지만 무거움 |
| 적용 | 동적 상태 추정 | 정적 매핑, 잔차 모델링 |

→ EKF와 GPR은 **상보적**: EKF는 동적 상태 추정, GPR은 모델 불확실성/잔차 보상에 각각 강점.

## 언제 사용하는가?

### 적합한 상황
- ✅ 물리 모델이 있지만 **파라미터 불확실성이 크거나 시변**인 경우 (배터리 노화, 타이어 마모)
- ✅ 안전 보장이 필요한 제어 — 불확실성 상한을 제약에 반영해야 할 때
- ✅ 데이터가 제한적이어서 신경망보다 **소량 데이터에서 잘 일반화**되어야 할 때
- ✅ 모델의 신뢰도 자체를 출력으로 사용해야 할 때

### 주의가 필요한 상황
- ⚠️ **대용량 데이터셋**: O(n³) 학습 복잡도 → Sparse GPR(유도점) 필요
- ⚠️ 고차원 입력: 커널 행렬 크기가 폭발 → 차원 축소 전처리 필요
- ⚠️ 실시간 전체 업데이트가 필요한 경우 어려움 (Online Sparse GPR으로 완화 가능)
